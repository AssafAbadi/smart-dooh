# Display / Billboard – reference for debugging

**State as of:** Working setup (white text, fade only on ad change, live mirror of app).

## What it does

- **URL:** `http://<BACKEND_IP>:3000/display/<driverId>` (e.g. `/display/driver-1`).
- **Behavior:** The page shows the **same ad the app last received** for that driver. It polls the backend every 5s and updates only when the ad content changes (no fade if same ad). White headline/coupon on black, gray body.

## How it works

1. **App** (or any client) calls `GET /ad-selection/ranked` (or `GET /ad-selection/ranked/:driverId`) with `driverId` + location.
2. **Backend** returns ranked instructions and **caches** the result in Redis under `display:last:<driverId>` (TTL 60s). See [ad-selection.controller.ts](apps/backend-api/src/ad-selection/ad-selection.controller.ts): `cacheLastForDisplay()`, `getLastForDisplay()`.
3. **Display page** is served by `GET /display/:driverId` (HTML generated in [display-html.ts](apps/backend-api/src/display/display-html.ts)). Its inline script polls `GET /ad-selection/last/:driverId` every 5s and renders the first instruction. It compares `headline|body|couponCode|businessId` to avoid re-running the fade when the ad is unchanged.

## Key files

| File | Role |
|------|------|
| [apps/backend-api/src/display/display-html.ts](apps/backend-api/src/display/display-html.ts) | Generates full HTML/CSS/JS for the billboard (colors, fade logic, same-ad check). |
| [apps/backend-api/src/display/display.controller.ts](apps/backend-api/src/display/display.controller.ts) | Serves `GET /display/:driverId` by calling `getDisplayHtml(driverId)` and sending `text/html`. |

## UX details (current)

- **Colors:** Headline and coupon = white (`#fff`), body = light gray (`#e0e0e0`), background black.
- **Animation:** Fade-in only when the ad **changes** (comparison key: headline + body + couponCode + businessId). Same ad → no fade.
- **Polling:** 5 seconds.

**Real location in the app:** If location is denied in Expo Go (iOS), use a development build (`npx expo run:ios` or EAS) so the app has its own Info.plist and real GPS works.

## Common bugs / fixes

- **Display still shows old colors or old behavior after code change:** The HTML is generated by the **running** backend. Restart the backend so it loads the new code; browser reload alone is not enough.
- **“No ad” or “Mirror – Open the app”:** Display uses cached “last” result. Ensure (1) Redis is running, (2) the **app** (or something) has called `GET /ad-selection/ranked` for the same `driverId` at least once so the cache is filled. Use same driverId in the URL as in the app (e.g. `driver-1`).
- **Display not updating when app changes ad:** Cache TTL is 60s; display polls every 5s. If the app requests ranked ads, the new result is cached and the display will show it on the next poll. If it doesn’t update, check that the app is actually calling the ranked API with that driverId and that Redis is up.

## Network

- Backend must listen on `0.0.0.0` and CORS enabled (`origin: true`) so the display URL works from other devices on the same Wi-Fi (e.g. phone or second PC).
